### Vercel + Resend konfiguration för kontaktformulär

För att aktivera e-postfunktionaliteten i kontaktformuläret behöver du:

1. **Skapa ett konto på [Resend](https://resend.com/)** (gratis plan finns)
2. **Hämta din API-nyckel** från Resend Dashboard
3. **Verifiera din domän** i Resend (eller använd deras testdomän för utveckling)
4. **Lägg till miljövariabler i Vercel:**
   - Gå till ditt Vercel-projekt → Settings → Environment Variables
   - Lägg till:
     - `RESEND_API_KEY` - Din Resend API-nyckel
     - `CONTACT_EMAIL` - Din e-postadress där meddelanden ska skickas (t.ex. "studypro.uf@email.com")

5. **För lokal utveckling**, skapa en `.env.local` fil i projektets rot:
```
RESEND_API_KEY=your-resend-api-key
CONTACT_EMAIL=studypro.uf@email.com
```

**Viktigt:** 
- För produktion måste du verifiera din domän i Resend för att kunna skicka från din egen domän
- Under utveckling kan du använda Resend's testdomän `onboarding@resend.dev`
- Efter verifiering kan du ändra `from`-adressen i `api/send-email.ts` till din egen domän

Create a `settings` table in Supabase to manage hero-texter och knappar:

```sql
create table if not exists public.settings (
  id bigint generated by default as identity primary key,
  hero_title text,
  hero_subtitle text,
  primary_cta_label text,
  secondary_cta_label text
);

insert into public.settings (hero_title, hero_subtitle, primary_cta_label, secondary_cta_label)
values (
  'Nå dina studiemål med StudyPro-UF',
  'Få personlig studiehjälp från erfarna studiebuddies inom matematik, fysik och mer. Vi hjälper dig att lyckas.',
  'Kontakta oss idag',
  'Läs mer om oss'
) on conflict do nothing;
```

The Supabase client is defined in `src/lib/supabaseClient.ts` and consumed via `fetchSettings` in `src/lib/settings.ts`. `src/components/Hero.tsx` uses these settings with a graceful fallback when env vars are missing.
